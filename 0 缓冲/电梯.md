# 方案一
1、增加CNPS(最近梯优先服务)功能：该功能（即优先指派物理距离最近且方向匹配的电梯响应呼叫）是一个常见的优化策略，但它是一把双刃剑。以下是其利弊的详细分析，
## 核心优势（利）
①、降低平均等待时间（尤其在低峰期或均匀分布呼叫时）。
②、当呼叫点附近正好有静止或方向匹配的空闲电梯时，它能最快到达。对于离电梯厅较远的楼层呼叫，或当电梯分布较分散时，效果更明显，乘客能更快地等到电梯。
③、避免从较远位置调派电梯过来响应呼叫，节省电梯运行能耗，减少机械磨损，理论上延长设备寿命，并降低运行成本。
④、乘客直观上认为“最近的电梯就该来接我”，就近响应符合这种心理预期。减少乘客因看到“明明有更近的电梯却不动”而产生的抱怨和焦虑，提升用户体验。
## 核心劣势（弊）
①、计算物理距离和方向匹配度相对直观，算法复杂度可能低于预测全局最优调度（如综合考虑未来呼叫）。
②、可能导致“扎堆”现象（尤其在高峰期），当多个呼叫集中在某一区域时，附近所有电梯都被调去响应这些呼叫，导致其他区域（特别是呼叫密度低的区域）的呼叫长时间无人响应。整体系统效率下降，部分乘客等待时间反而显著增加。这是就近响应策略在高峰期的最大弊端。
③、忽视电梯负载和未来需求预测：只关注当前呼叫和电梯当前位置/方向，不考虑电梯当前载客量（满载电梯即使最近也无法响应新呼叫）或预测即将到来的呼叫。
④、可能导致满载错过：派出的“最近”电梯满载，无法接客，造成无效运行和额外等待。为了响应一个近但方向不匹配的呼叫，电梯可能需要空跑一段或改变方向，打乱原本更优的运行计划，反而影响后续多个呼叫的响应效率。
⑤、乘客等待时间短了，但乘梯时间可能变长，整体旅程时间未必最优。
# 方案二
2、增加CNPS(最近梯优先服务)、PRS(防止同时运行)、UCPS(轻载梯优先服务)：在原有就近响应电梯基础上额外增加优先指派当前载重率较低（可设置：如低于30%）且满足方向条件的电梯响应新呼叫，通过算法错开多部电梯的加速启动，避免多部电梯在同一瞬间从静止状态启动。以下是其利弊的详细分析
## 核心优势（利）
①、轻载电梯轿内指令相对更少，停靠楼层耗时更短，能更快到达呼叫层。缩短乘客等待时间，尤其对短距离呼叫（如相邻楼层）效果明显。
②、避免“无效响应”， 防止派出一部即将满载的电梯（即使它最近），结果因满载而无法接客。减少电梯空跑，提升调度成功率。
③、缓解高峰期瓶颈，提升高峰期系统吞吐量。
④、轻载电梯运行能耗远低于重载电梯，优先调度可降低整体系统能耗。减少电费成本，符合绿色建筑标准。
## 核心劣势（弊）
①、若持续有新的呼叫，轻载电梯可能被不断指派新任务，而重载电梯始终无法完成任务释放载重，导致系统负载不均。重载电梯服务区域乘客等待时间大幅延长。
②、增加长距离乘客乘梯时间， 轻载电梯可能因频繁响应短途呼叫而频繁启停，导致轿厢内长途乘客旅程时间增加。
③、为响应一个轻载机会，电梯可能中途改变方向或插入短途任务，打乱原有高效运行路径。
# 方案三
3、闲时就近梯优先服务，早晚高峰设置分区运行。 
## 核心优势（利）
①、避免高峰期运行就近响应带来的弊端。
②、电梯在固定区间运行，减少跨区长距离移动和停靠次数。减少电梯“被拦截”现象，避免“层层停”的烦躁体验。
③、 减少长距离跨区空跑和频繁启停。  
## 核心劣势（弊）
①、楼层分配不均引发公平性问题，比如“边缘楼层”困境：例如12层被划入低区，但13层乘客需等待专属高区电梯（即使低区电梯空闲）。
②、用户投诉风险：高区乘客在晚高峰可能等待过久
③、若实际客流与预设分区不匹配（如某日大量员工前往中区会议室），系统无法自动调整，效率反而低于自由调度。
# 现行方案
群控（3部及其以上即为群控）采用的是全局分配方式，考虑整个大楼中的情况选取最优电梯，并不是像并联（两部电梯为并联）一样就近响应。正在待机电梯的响应优先级较低。（比如客梯先进入待机模式了，货梯还没有进入待机模式，虽然客梯距离呼梯楼层近，他的响应优先级也没得货梯高）。群控并不是每时每刻根据当前的电梯距离调整分配的，因为若频繁地更改群控分配，会大幅影响群控运行效率。只有等待超过一定时间后此前分配的电梯仍无法响应时，才会换另一台去响应 。
